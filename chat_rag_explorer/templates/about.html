About RAG Lab

Purpose

RAG Lab is an educational application designed to demonstrate the implementation of a chat interface with Large Language Models (LLMs) and Retrieval-Augmented Generation (RAG).

Interested in a RAG workshop for your team? Contact info@alteredcraft.com. See past workshop deliveries for examples.

Technology Stack

  • Backend: Flask (Python)
  • LLM Access: OpenRouter (GPT-4, Claude, Llama, etc.)
  • Vector Database: ChromaDB
  • Frontend: Vanilla JavaScript (no framework)

Features

Inspect Request Details

Click "view details" on any message to see exactly what the LLM receivedâ€”model, parameters, token counts, timing, and retrieved RAG documents with their source metadata and similarity scores.

Real-time Streaming

Server-Sent Events (SSE) stream LLM responses token-by-token for a responsive chat experience.

Model Selection

Dynamic model picker with OpenRouter models, filtered to RAG-recommended options.

Conversation History

Multi-turn conversation support with full context retention.

Session Metrics

Real-time token usage tracking for both individual interactions and session totals.

RAG Integration

Retrieve relevant documents from ChromaDB and inject them as context for the LLM.

Markdown Rendering

Secure rendering using Marked.js and DOMPurify, bundled locally for offline use.

Architecture Highlights

  • Application Factory Pattern: Flask best practices for modular, testable code
  • Singleton Services: Chat, RAG config, prompts, and history services
  • Request ID Correlation: All API endpoints include unique IDs for log tracing
  • ChromaDB Modes: Supports local files, server connections, and cloud deployment

Getting Started

RAG Lab requires Python 3.11+ and an OpenRouter API key. A sample ChromaDB with pre-embedded documents is included to get started quickly.

For full setup instructions and documentation, see the project's README file.
